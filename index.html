Example Displayer

A Displayer is rendered by combining .js/.json sources. The steps are as follows:

  1. Ask the client library for a displayer for an InterMine object.
  2. The library loads the data (JSON), templates (eco), presenter ("how should data be rendered") while providing a callback for when these are returned from the server.
  3. Once done, the presenter is initialized with a bunch of options like imObj, displayer name, data payload etc.
  4. Compiled CoffeeScript eco templates are just JavaScript functions that can be called with the payload.
  5. As presenters are arbitrary objects with functions, any kind of behavior can be encompassed.
 

backbone.js Displayer

This time, the presenter returned extends Backbone.View.

Notice how the data and templates can be used without changes, while eco templates can be cached for future calls.
